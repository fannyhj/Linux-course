# x) Tiivistelmä “Pro Git 1.3 – What is Git?”

- Git tallentaa muutokset snapshotteina, ei diff-tiedostoina.
  
- Gitissä melkein kaikki toiminnot tapahtuvat paikallisesti, mikä tekee käytöstä nopeaa ja turvallista.

- Jokainen commit on kryptografisesti tunnisteella suojattu (SHA-1).


## Git add, commit, pull & push 

1. git add .

- Lisää kaikki työskentelyhakemistossa muuttuneet tiedostot staging-alueelle.

- Lähde esim: Pro Git – Git Basics.
  

2. git commit

- Luo uuden commitin paikalliseen repositoryyn.

- Vaatii yleensä commit-viestin esim. git commit -m "viesti".
  

3. git pull hakee palvelimelta (esim. GitHub) uudet commitit ja yhdistää ne paikalliseen työhön.

- Vastaa käytännössä: git fetch + git merge.
  

4. git push
   
- Lähettää paikalliset commitit palvelimelle.


Lähde: Git Cheat Sheet (https://git-scm.com/cheat-sheet)
